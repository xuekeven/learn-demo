# 小红书评价采集

这个 skill 负责把小红书搜索结果整理成可用于选购决策的商品评价报告。

## 触发条件

- 用户以 `/get-red-book-evaluate` 开头发起请求
- 用户用大意类似“总结小红书用户对 `${商品A}` 的评价”的句子提出需求

## 执行方式

1. 按顺序读取 `prompts/` 里的工作流步骤文件。
2. 每完成一个步骤，都确认该步骤的“完成标准”已经满足。
3. 必须使用 Codex 的 `chrome:control-chrome` 插件接管 Google Chrome。
4. 必须先做小红书首页、登录态、搜索能力的前置检查；任何时候异常时立即停止并询问用户下一步决策。
5. 用新标签页打开每个帖子，最终保留所有已打开的帖子标签页，并在报告中留下每篇帖子的 URL，方便回看。
6. 采集帖子时模拟人类阅读节奏：每个帖子至少停留 5 秒，用于判断时间/IP、广告属性、正文价值和评论价值。

## 工作流步骤

- `prompts/00-overview.md`：整体任务目标与产出要求
- `prompts/01-clarify.md`：澄清输入信息、时间/IP筛选条件
- `prompts/02-preflight.md`：Chrome 与小红书可用性检查
- `prompts/03-search.md`：搜索、候选帖收集与有效帖筛选
- `prompts/04-evaluate-post.md`：新标签页打开帖子并评估正文与评论
- `prompts/05-write-report.md`：写入 Markdown 报告
- `prompts/06-summarize.md`：综合结论与购买建议
- `templates/report-template.md`：最终 Markdown 总结报告模板

## 约束

- 默认目标是 8 个“有效贴”，不是 8 个搜索结果
- 有效贴必须不是广告贴，并且在用户指定时符合发布时间/修改时间和 IP 范围
- 如果帖子正文和评论都没有购买参考价值，也不算有效贴
- 每个帖子至少查看 5 条评论；评论多时最多查看 50 条，优先高赞和高回复评论
- 遇到折叠评论或折叠回复时，必须展开后再判断
- 只提取和购买决策相关的信息
- 区分正文观点和评论观点
- 对单条评论推断保持谨慎
- 任何时候页面无法加载、未登录、白屏、验证码阻塞或无法搜索，立即停止，报告原因并要求用户决定下一步
- 不私信、不评论、不联系用户、不提交任何表单
