# 06 - 综合结论与购买建议

## 目标

把多篇有效帖和评论综合成明确的购买建议。

## 输入

- Markdown 报告草稿
- 每篇有效帖的购买信号
- 跳过帖子列表及原因

## 动作

汇总以下维度：

- 共同正面点
- 共同风险点
- 价格区间
- 配件与加装争议
- 续航与安全性
- 售后与合规风险
- 时间/IP 范围对结论的影响

## 最终输出

给出明确建议，使用下面三种表达之一：

- `可以买，但要满足...`
- `谨慎买，除非...`
- `不建议买，如果你在意...`

## 必须补充

- 说明一共看了几个有效帖。
- 说明结论主要来自正文还是评论。
- 说明是否应用了时间/IP过滤。
- 说明跳过了哪些类型的帖子。
- 列出付款前要检查的具体项目。

## 产出

- 最终购买建议
- 付款前检查清单

## 完成标准

- 用户能明确知道该不该买。
- 用户能明确知道应该按哪些条件砍价、验货、避坑。
- 所有有效帖标签页最终保留打开。
