# 03 - 搜索、候选帖收集与有效帖筛选

## 目标

在小红书搜索目标商品，并按页面视觉顺序筛选出目标数量的有效帖。

## 输入

- 商品关键词
- 目标有效帖数量，默认 8 个
- 时间范围过滤条件，可能为空
- IP 范围过滤条件，可能为空

## 动作

1. 在小红书首页搜索框输入关键词并提交。
2. 确认搜索结果页确实进入对应关键词。
3. 按页面视觉顺序读取候选帖子：从左到右、从上到下。
4. 对每个候选帖子，先记录标题、作者/日期（如果可见）、URL 和结果顺序。
5. 逐个用新标签页打开候选帖子做有效性判断。
6. 符合条件的帖子计入有效帖；不符合的跳过并记录原因。
7. 如果当前页有效帖不足，继续滚动搜索结果页加载更多候选帖。

## 有效帖判定

一个帖子只有同时满足以下条件才算有效帖：

- 不是广告贴、推广贴、软广或明显带货引流帖
- 帖子链接和已收集帖子不重复
- 如果用户指定了时间范围，能看到发布时间或修改时间，且帖子时间在范围内
- 如果用户指定了 IP 范围，能看到 IP 地址，且帖子 IP 在范围内

## 广告贴识别

遇到以下特征时跳过，并记录为疑似广告：

- 明确标注广告、赞助、合作、推广
- 正文主要引导购买、私信、加微信、进群、报暗号
- 内容高度模板化，只夸优点不提真实使用细节
- 评论区明显由引流回复组成
- 帖子主体是商家、门店、代购或销售线索

## 产出

- 有效帖列表
- 每个有效帖的标题、URL、发布时间/修改时间、IP、结果顺序
- 跳过帖子列表及跳过原因

## 完成标准

- 已获得目标数量的有效帖，或已经说明为什么不足目标数量。
- 每个有效帖都有唯一 URL。
- 每个有效帖都保留在新标签页中，不要关闭。

## 注意事项

- 不要按 DOM 原始顺序猜测视觉顺序。
- 如果页面进入 AI 搜索或普通搜索结果不同布局，以实际可见卡片为准。
- 有效帖数量指过滤后的帖子数量，不是打开过的帖子数量。
