# 05 - 写 Markdown 报告

## 目标

把采集结果写成结构稳定、可读、可追溯的 Markdown 文件。

## 输入

- 有效帖列表
- 跳过帖子列表及原因
- 正文摘要
- 评论分类要点
- 页面限制说明
- 输出路径

## 动作

使用 `templates/report-template.md` 作为最终报告结构。按固定结构写入每篇有效帖：

1. 帖子标题
2. 帖子 URL
3. 发布时间或修改时间
4. IP 地址
5. 正文摘要
6. 正面评论要点
7. 负面评论要点
8. 中性/参考评论要点
9. 该帖子的购买信号
10. 已查看评论数量

报告末尾补充：

- 有效帖数量
- 跳过帖子数量与原因，例如广告、时间不符、IP 不符、信息缺失
- 采集限制，例如折叠评论、加载失败、无法展开楼中楼
- 每个有效帖至少停留 5 秒的采集说明
- 无价值帖、广告帖、时间/IP不符帖的跳过统计

## 写作要求

- 用中文。
- 简短清晰。
- 不要大段照抄原文。
- 区分正文和评论。
- 标明哪些结论只是单条评论观点。
- 每篇有效帖必须保留 URL。
- 不要把“打开过的帖子数”写成“有效帖数”。

## 产出

- Markdown 报告草稿

## 完成标准

- 每篇有效帖都有独立章节。
- 每篇都有唯一 URL，方便用户回看。
- 每篇都有发布时间/修改时间和 IP。
- 每篇都有已查看评论数量。
- 报告没有把单条评论包装成确定事实。
