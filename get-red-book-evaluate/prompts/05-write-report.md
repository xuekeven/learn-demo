# 05 - 写 Markdown 报告

## 目标

把采集结果写成结构稳定、可读、可追溯的 Markdown 文件。

## 输入

- 有效帖列表
- 跳过帖子列表及原因
- 正文摘要
- 评论分类要点
- 页面限制说明
- 采集日志
- 输出路径

## 动作

使用 `templates/report-template.md` 作为最终报告结构。按固定结构写入每篇有效帖：

1. 帖子标题和 URL 写在同一个 Markdown 超链接标题中，例如 `### 1. [帖子标题](帖子完整URL)`
2. 时间
3. IP 地址
4. 正文表格：列名为“用户名、证据片段、总结要点、评价”
5. 评论表格：列名为“用户名、证据片段、总结要点、评价”
6. 该帖子的购买信号
7. 已查看评论数量

报告末尾补充：

- 候选帖数量和打开帖数量
- 有效帖数量
- 跳过帖子数量与原因，例如广告、时间不符、IP 不符、信息缺失
- 停止采集原因：已达到目标有效帖数量，或累计打开 20 个候选帖后有效帖仍不足
- 采集限制，例如折叠评论、加载失败、无法展开楼中楼
- 标签页批处理说明：是否曾因单批候选帖较多而关闭中间候选帖标签页；如果关闭过，说明有效帖原始 URL 已保留，可用于回看或最终重开
- 每个有效帖至少停留 5 秒的采集说明
- 无价值帖、广告帖、时间/IP不符帖的跳过统计

## 写作要求

- 用中文。
- 简短清晰。
- 不要大段照抄原文。
- 只保留短摘录或证据片段，不复制长段正文或评论。
- 正文和评论分成两个独立表格展示，表格标题分别为“正文：”和“评论：”。
- 正文表格和评论表格的列名都必须是“用户名、证据片段、总结要点、评价”。
- 表格中的“用户名”填写正文作者或评论作者；无法稳定识别时写“未识别”，不要猜。
- 标明哪些结论只是单条评论观点。
- 每篇有效帖必须保留 URL，并且标题和 URL 必须合并为超链接形式，不再单独写 `URL：...`。
- URL 使用从页面获得的原始链接，不手动改写、清洗或短链化。
- 如果中途因批次标签页管理关闭过有效帖标签页，报告仍必须使用关闭前记录的原始 URL。
- 报告顶部采集元信息使用两列表格展示，列名为“项目”和“内容”。
- 每篇有效帖的元信息使用两列表格展示，列名为“项目”和“内容”，包含：时间、IP、是否疑似广告、已查看评论数量、有效性判断。
- 不使用旧的复合时间字段名，统一写成“时间”。
- 不要把“打开过的帖子数”写成“有效帖数”。
- 已查看评论数量按“主评论 + 展开回复逐条计数”填写。

## 产出

- Markdown 报告草稿

## 完成标准

- 每篇有效帖都有独立章节。
- 每篇标题都是可点击的 Markdown 超链接，链接为唯一 URL，方便用户回看。
- 每篇都有时间和 IP。
- 每篇都有已查看评论数量。
- 报告没有把单条评论包装成确定事实。
