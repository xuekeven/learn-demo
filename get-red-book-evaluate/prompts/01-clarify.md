# 01 - 澄清输入

## 目标

只澄清执行任务所必需的信息，并从用户请求中提取筛选条件。

## 输入

- 用户原始请求

## 动作

1、提取

从用户请求的描述中：
- 提取商品关键词。
- 提取是否指定有效帖数量。
- 提取是否指定时间范围。
- 提取是否指定 IP 范围。
- 提取是否指定输出路径。
- 如果用户指定有效帖数量，将其规范到 5-15 篇范围内：少于 5 按 5，多于 15 按 15。

2、确认

将上一步提取出来的内容向用户提问，确认五个参数是否正确。提取出来的“商品关键词”不可为空，其余可为空。

- “商品关键词”不为空时，使用模版1提问。如果用户回复“确认”之类的认可语句，进入下一步；否则，综合用户所有的回复，回到“1、提取”这步骤。
- “商品关键词”为空时，使用模版2提问。针对用户的回复，回到“1、提取”这步骤。

模版1：

```
收到，总结如下：

**商品关键词**：${商品关键词}
**有效帖数量**：${有效帖数量}
**时间范围**：${时间范围}
**IP 范围**：${IP 范围}
**总结输出路径**：${总结输出路径}

确认吗，可以的话，我直接开始下一步。
```

模版2：

```
针对您刚刚的输入，我未能提取到商品关键词这个关键信息，请您直接复制以下内容，填写后发给我。

**商品关键词（必填）**：
**有效帖数量**：
**时间范围**：
**IP 范围**：
**总结输出路径**：

```

## 默认值

- 有效帖数量默认 8 个；用户可指定 5-15 个。
- 如果用户指定少于 5 个，有效帖数量按 5 个执行，并在确认里说明“已按最少 5 篇处理”。
- 如果用户指定多于 15 个，有效帖数量按 15 个执行，并在确认里说明“已按最多 15 篇处理”。
- 如果用户不指定时间范围，不按时间过滤。
- 如果用户不指定 IP 范围，不按 IP 过滤。
- 如果用户不指定输出路径，默认写到工作区文件。

## 产出

- 商品关键词
- 目标有效帖数量
- 时间范围过滤条件，可能为空
- IP 范围过滤条件，可能为空
- 输出路径

## 完成标准

- 已经具备可开始打开小红书并搜索的最小输入。

## 注意事项

- 不要追问过多格式细节。
- 即使用户目标已经足够明确，也要先确认提取出的五个参数。
