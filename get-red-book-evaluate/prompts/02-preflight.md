# 02 - Chrome 与小红书前置检查

## 目标

确认 Chrome、小红书首页、登录态和搜索能力都正常。全局阻塞时立即停止，不进入采集流程。

## 输入

- 商品关键词

## 动作

1. 使用 Codex 的 `chrome:control-chrome` 插件连接并接管 Google Chrome。
2. 在新页签打开 `https://www.xiaohongshu.com/`。
3. 检查页面是否正常加载。
4. 检查是否处于已登录状态。
5. 检查是否能看到并操作搜索框。
6. 可用目标关键词或轻量方式验证搜索入口确实可用。

## 异常处理

遇到以下全局阻塞时，立即停止：

- Chrome 插件无法连接
- 小红书网页打不开
- 页面白屏或长时间加载失败
- 未登录，无法搜索或无法访问内容
- 出现验证码、安全验证或其他阻塞
- 搜索框不可见或不可操作

停止时向用户报告：

- 具体卡在哪一步
- 看到的页面状态
- 建议用户选择下一步，例如登录、刷新、换网络、稍后重试

以下情况不视为前置检查失败，可在后续采集中记录限制并继续：

- 个别帖子打不开
- 个别评论区加载失败
- 个别折叠评论或楼中楼无法展开
- 搜索结果中出现无关、重复或广告内容

## 产出

- 前置检查通过或失败的结论
- 失败时的原因说明

## 完成标准

- 只有确认“小红书正常已登录且能搜索”后，才进入下一步。
