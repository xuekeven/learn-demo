# 02 - Chrome 与小红书前置检查

## 目标

确认 Chrome、小红书首页、登录态和搜索能力都正常。全局阻塞时立即停止，不进入采集流程。

## 输入

- 商品关键词

## 动作

1. 使用 Codex 的 `chrome:control-chrome` 插件连接并接管 Google Chrome。
2. 使用 `chrome:control-chrome` 创建新的受控 Chrome 标签页打开 `https://www.xiaohongshu.com/`。
3. 前置检查和采集必须在 `chrome:control-chrome` 创建或接管的独立受控标签页中完成；不要复用用户正在操作的标签页。
4. 如果需要隔离采集现场，优先使用插件创建的受控标签页；不要为了打开系统级新窗口而改用 `computer-use`、`osascript` 或手动点击 Chrome 菜单。
5. 若用户明确要求系统级新窗口，则说明 `chrome:control-chrome` 常规 API 不直接提供该能力，并请用户手动打开新窗口后，再用 `chrome:control-chrome` 接管其中的标签页。
6. 检查页面是否正常加载。
7. 检查是否处于已登录状态。
8. 检查是否能看到并操作搜索框。
9. 可用目标关键词或轻量方式验证搜索入口确实可用。

## 异常处理

遇到以下全局阻塞时，立即停止：

- Chrome 插件无法连接
- 小红书网页打不开
- 页面白屏或长时间加载失败
- 未登录，无法搜索或无法访问内容
- 出现验证码、安全验证或其他阻塞
- 搜索框不可见或不可操作

如果 Chrome 插件无法连接，报告“Chrome 插件无法连接”，不要自动改用其它浏览器、`computer-use`、独立 Playwright 或普通网页搜索采集小红书内容。

停止时向用户报告：

- 具体卡在哪一步
- 看到的页面状态
- 建议用户选择下一步，例如登录、刷新、换网络、稍后重试

以下情况不视为前置检查失败，可在后续采集中记录限制并继续：

- 个别帖子打不开
- 个别评论区加载失败
- 个别折叠评论或楼中楼无法展开
- 搜索结果中出现无关、重复或广告内容

## 产出

- 前置检查通过或失败的结论
- 失败时的原因说明

## 完成标准

- 只有确认“小红书正常已登录且能搜索”后，才进入下一步。
