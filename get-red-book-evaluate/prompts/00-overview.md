# 00 - 总体目标

## 目标

把小红书搜索结果整理成一份可以直接辅助购买决策的 Markdown 报告。

## 输入

- 商品关键词
- 有效帖数量，默认 8 个，允许范围 5-15 个
- 可选时间范围，例如“近一个月”“2026-03-01 到 2026-06-01”
- 可选 IP 范围，例如“只看北京/上海 IP”
- 输出路径，默认工作区 Markdown 文件

## 动作

- 使用 `chrome:control-chrome` 接管 Google Chrome。
- 新页签打开 `https://www.xiaohongshu.com/` 并完成前置检查。
- 搜索用户给出的商品关键词。
- 按视觉顺序筛选候选帖子，直到得到目标数量的有效帖，或累计打开 20 个候选帖后停止新增候选帖。
- 每篇帖子都用新标签页打开。
- 拿到目标数量的有效帖后立即停止打开新候选帖，不要求打开满 10 个页签。
- 单批最多同时打开 10 篇候选帖；记录完成后关闭中间候选帖标签页，避免标签页过多导致电脑卡顿。
- 每篇都先看正文，再看评论。
- 只保留和选购相关的有价值信息。
- 最后给出综合购买建议。

## 有效帖定义

有效帖必须同时满足：

- 是真实商品讨论帖
- 不是明显广告贴或暗广
- 如果用户指定时间范围，能看到帖子时间，帖子时间符合要求
- 如果用户指定 IP 范围，能看到帖子 IP 地址，帖子 IP 符合要求

## 产出

- 一份 Markdown 报告。
- 每篇有效帖单独成节。
- 每篇都写明标题、URL、时间、IP、正文要点、评论要点。
- 评论按正面、负面、中性/参考整理。
- 结尾包含跨帖子综合结论。

## 完成标准

- 用户能从报告中看出“是否值得买、为什么、付款前要检查什么”。
- 报告中明确说明查看了多少个有效帖，以及跳过了哪些类型的帖子。
- 报告中明确说明目标有效帖数量、实际有效帖数量、累计打开候选帖数量，以及是否因为达到 20 个候选帖上限而停止新增候选帖。
- 报告中明确说明是否关闭过中间候选帖标签页，并保留每篇有效帖原始 URL 方便回看。
